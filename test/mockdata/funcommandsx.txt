                				<b><font size="5">FuncommandsX</font></b> - By Spazman0 and Arg!<br />
<br />
Description: Additional fun commands for SourceMod. Each fun command has full admin menu integration. Built and tested for SourceMod <b>1.6.2</b> using TF2, DOD:S and CS:S. Joint project, our first release.<br />
<b><br />
Admin Commands: </b><ul><li>sm_alpha - sets player alpha (visibility) 0 is invisible, 255 is opaque (normal)</li>
<li>sm_colorize - sets player color (normal to revert)</li>
<li>sm_fakesay - forces a client to say text, as if they typed it in chat (ROOT admin only, override to give other admins access)</li>
<li>sm_ff - toggles friendly fire (this is triggered from the Server Commands admin menu)</li>
<li>sm_happy - replaces clients text chat with strings from happy_phrases.ini</li>
<li>sm_invis - toggles or sets player invisibility</li>
<li>sm_invisplayer - toggles or sets player invisibility (does not affect player weapon)</li>
<li>sm_tele - Teleports player to where admin is looking</li>
<li>sm_explode - Explodes player(s) (only works with games supporting the 'explode' client console cmd, eg TF2)</li>
<li>sm_disco - changes all players colors to random every x seconds (this is triggered from the Server Commands admin menu). x is defined by cvar sm_discointerval.</li>
</ul><b>Console Commands:<br />
</b><ul><li>sm_me - performs an IRC style 'me' action.</li>
<li>sm_colorize_colors - displays colors available to sm_colorize</li>
<li>sm_colorme &lt;color&gt; - allows players to colorize themselves.</li>
</ul><b>Cvars:</b><ul><li>sm_discointerval - interval in seconds for how often players colors will change in disco mode. Default is 0.6 seconds.</li>
<li>sm_autodisco         &lt;1|0&gt; - If enabled, discomode is enabled on map change.</li>
<li>sm_autodiscotime    Time in seconds to run auto disco mode for, default 60.0</li>
<li>sm_sa_me &lt;1|0&gt; - Toggles availability of sm_me command, defaults to on.</li>
<li>sm_selfcolorize &lt;1|0&gt; - Toggles ability to use sm_colorme command</li>
</ul><b>Other:</b><br />
<br />
To change the message displayed when disco mode is activated edit the 'funcommandsX.phrases.txt' file and change the text in the 'Disco on message' section.<br />
<br />
Translation support for some phrases added as of version 1.4. If you are able to add any translations please feel free to let <a href="https://forums.alliedmods.net/member.php?u=39162" target="_blank" rel="noopener">Arg!</a> know, it would be appreciated. Also if you find any phrases which need to be setup for translation, let <a href="https://forums.alliedmods.net/member.php?u=39162" target="_blank" rel="noopener">Arg!</a> know as well.<br />
<br />
Requires colors.inc to compile - <a href="http://forums.alliedmods.net/showthread.php?t=96831" target="_blank" rel="nofollow noopener">http://forums.alliedmods.net/showthread.php?t=96831</a><br />

<li>bl4nk</li>
</ul><b>Install Instructions:</b><ul><li>plugins/funcommandsX.smx goes into 'sourcemod/plugins'</li>
<li>configs/happy_phrases.ini goes into 'sourcemod/configs' and can be edited to allow for custom 'happy' phrases.</li>
<li>The file 'funncommandsX.cfg' will be created in 'cfg/sourcemod' and can be used to change default cvar settings.</li>
<li>The file 'translations/funcommandsX.phrases.txt' must go in the translations directory 'addons/sourcemod/translations'</li>
</ul><b>Known Issues:</b><ul><li>Team Fortress 2 - If a player has been colorized, made invisible or had their alpha changed, players using the spy radio command on them will not hear the class specific sound file, instead a generic 'spy' sound. This resets for a player on map change or if they reconnect. This (<a href="http://forums.alliedmods.net/showthread.php?t=105646" target="_blank" rel="nofollow noopener">http://forums.alliedmods.net/showthread.php?t=105646</a>) plugin has been developed to counter it if you are interested in it.</li>
